# Getting Ralph VC onto your iPhone

There are two click-to-install paths. Pick the one that matches what
you have on hand right now.

| Path | What you need | What you get | Set-up time |
| --- | --- | --- | --- |
| **A — PWA (zero setup)** | An iPhone with Safari. That's it. | Works today. Lower-fidelity speech (Web Speech API). No keychain — bearer token in localStorage. | 30 seconds |
| **B — Ad-Hoc OTA install** | $99/yr Apple Developer Program, a Mac with Xcode, your iPhone's UDID registered, an HTTPS host (free options below). | Real native iOS app. Native `SFSpeechRecognizer` STT, native `AVSpeechSynthesizer` TTS with the Ralph voice. Tap a Safari link → iOS prompts to install. | ~30 min one-time |

> A third path — **TestFlight** — gives the cleanest install experience
> (tap an emailed link → install from Apple's TestFlight app) but adds
> Apple's beta review (typically same-day). Same prerequisites as path B
> plus App Store Connect setup. Ask if you want me to wire this up.

## Path A — PWA, zero setup

1. On your Mac (only this once), serve the `web/` directory:

   ```bash
   cd web/
   python3 -m http.server 8080
   ```

   …or push the contents of `web/` to **any** static host (GitHub Pages,
   Netlify Drop, Cloudflare Pages, Vercel, etc.).

2. On your iPhone, open Safari and visit:

   - `http://<your-mac-ip>:8080/` (same Wi-Fi as the Mac), **or**
   - the URL of wherever you uploaded `web/`.

3. Tap Safari's **Share** icon → **Add to Home Screen** → **Add**. The
   Ralph VC icon appears on your home screen. Tap it — it opens
   fullscreen like a native app, no browser chrome.

4. Tap the gear icon → paste:

   - **Endpoint:** `http://<your-mac-ip>:7878` (where `setup.sh` started
     the orchestrator).
   - **Bearer token:** the value `setup.sh` printed (or `RALPHVC_BEARER`
     from your shell).

5. Tap the mic and start vibe-coding.

**Limits of the PWA path:**

- iOS Safari's `SpeechRecognition` is supported but slightly higher
  latency than `SFSpeechRecognizer`. No on-device-only mode.
- TTS uses `speechSynthesis`. The Ralph voice IS available if installed
  on the device — the JS picks any voice whose name matches `/Ralph/i`,
  same rule as the native app.
- The bearer token sits in `localStorage`, not the iPhone Keychain.
  Treat the token as ephemeral; rotate by re-running `setup.sh`.

## Path B — Ad-Hoc OTA, real native install

This is the only Apple-sanctioned way to install an `.ipa` directly via
Safari on an iPhone you don't ship through the App Store. Apple
requires three things; all three are one-time setup:

1. **Apple Developer Program membership** ($99/yr).
   <https://developer.apple.com/programs/>
2. **Your iPhone's UDID** registered in *Devices* in the developer
   portal. Find the UDID in *Finder → connected iPhone → device summary*
   (right-click the value to copy).
3. **An HTTPS host** for the `.ipa` + `manifest.plist` + install page.
   `host.sh` supports three free options out of the box: Cloudflare
   Pages, Netlify, and GitHub Pages.

### One-command flow

On your Mac:

```bash
# 1. Build a signed Ad-Hoc .ipa
./distribution/make-ipa.sh --team-id ABCDE12345

# 2. Publish to your hosting provider of choice
./distribution/host.sh --provider cloudflare      # or netlify / gh-pages
```

`host.sh` prints a URL like:

```
https://ralphvc.pages.dev/
```

### What to do on your iPhone

1. Open the printed URL **in Safari** (other browsers cannot trigger
   the install prompt).
2. Tap the big blue **Tap to install on iPhone** button.
3. iOS pops a system dialog: *"<your domain> would like to install
   Ralph VC"*. Tap **Install**.
4. Watch the icon appear on your home screen with a progress ring.
5. **First launch only:** open *Settings → General → VPN & Device
   Management*, find your developer profile, tap *Trust*. Then return to
   Ralph VC — it'll launch.
6. Tap the gear icon → paste the same endpoint + token as above (this
   time stored in the iPhone Keychain).
7. Tap the mic.

### What's in the install page

The install page (`build/install.html`) is generated by
`distribution/render_install_page.py` and contains:

- An `apple-touch-icon` so the install prompt shows the Ralph VC logo.
- An `itms-services://?action=download-manifest&url=<HTTPS-manifest-url>`
  link — the only URL scheme iOS Safari will honour for OTA install.
- A QR code of the install URL so you can hand it to teammates whose
  iPhones are also UDID-registered on your developer account.
- Trust-the-developer instructions, since first-launch failure due to
  missing trust is the #1 gotcha.

### Re-issuing a build

Just re-run both scripts. `make-ipa.sh` bumps the build number to a
fresh timestamp; `host.sh` overwrites the published URL atomically so
your existing install link keeps working.

## Why I can't just give you a download link directly

Apple's iOS will **only** install an `.ipa` from Safari if all of the
following are true:

1. The `.ipa` is signed with a certificate that chains to Apple Root CA.
2. The signature includes a provisioning profile listing your iPhone's
   UDID (Ad-Hoc) **or** your enterprise team (Enterprise) **or** is
   downloaded through Apple's own services (App Store / TestFlight).
3. The install is initiated through `itms-services://` from an HTTPS
   page.

I can ship every line of script and HTML in this repo, but I cannot
sign the `.ipa` for you (no Apple Developer cert here), I cannot
register your UDID in your team (no portal access), and I cannot host
the result over HTTPS (no infra to attach to your account). The two
paths above are the closest I can practically get you to "tap and it's
installed" without taking over your developer account.
